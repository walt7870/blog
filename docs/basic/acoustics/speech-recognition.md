# 语音识别

语音识别 ASR 把语音信号转换成文本。它是语音助手、字幕、会议纪要、客服质检和语音输入法的核心模块。

## ASR 输入输出

输入通常是单声道 PCM 音频，常见采样率为 16 kHz 或 48 kHz。输出可以是纯文本，也可以包含时间戳、置信度、分词、标点、说话人信息和中间结果。

ASR 的结果不是只有一行文字。工程系统通常还会关心这句话从什么时候开始、什么时候结束、每个词的置信度是否可靠、是否还处在流式中间结果阶段。用户看到的是“把客厅空调调到二十六度”，系统内部还会把它交给意图识别、设备名匹配和参数提取。

## 基本结构

现代 ASR 通常包括：

| 模块 | 作用 |
| --- | --- |
| 特征提取 | 从波形中提取梅尔频谱、Filter Bank 等特征 |
| 声学模型 | 建模声音片段和文字单元之间的关系 |
| 解码器 | 根据模型输出搜索最可能文本 |
| 语言模型 | 提供词序和上下文约束 |
| 后处理 | 标点、数字规整、热词替换、敏感词处理 |

端到端模型把传统多模块压缩成一个统一模型，但工程上仍然需要热词、标点、文本规整和业务后处理。

## 流式与非流式

| 类型 | 特点 | 场景 |
| --- | --- | --- |
| 流式 ASR | 边说边出中间结果，延迟低 | 语音助手、实时字幕、会议 |
| 非流式 ASR | 拿完整音频后统一识别，准确率通常更高 | 录音转写、质检、离线批处理 |

流式识别需要处理“中间结果反复修正”。界面上应区分临时文本和最终文本，业务指令最好等最终结果或高置信片段再触发。

## 影响识别率的因素

| 因素 | 影响 |
| --- | --- |
| SNR | 噪声越大，语音特征越容易被淹没 |
| 混响 | 语音拖尾，辅音边界模糊 |
| 口音和方言 | 发音与训练数据差异大 |
| 领域词 | 人名、地名、型号、术语容易错 |
| 采样率和编码 | 低码率压缩会损伤细节 |
| 前处理 | 降噪过强会让语音失真 |
| 端点检测 | 截断开头或结尾会导致漏字 |

排查 ASR 错误时，要同时看原始录音和送入 ASR 的最终音频。很多错误来自前处理输出，而不是识别模型本身。

## 热词与领域词

热词用于提升特定词汇被识别出来的概率，例如联系人姓名、商品型号、业务术语、城市名。

使用建议：

- 热词列表要精简，避免把普通词挤掉。
- 同音词要配合上下文和业务规则处理。
- 动态热词适合联系人、设备名、歌名等个性化词表。
- 热词命中后仍要看置信度，不能无条件替换。

典型热词包括设备名、联系人姓名、商品型号、业务术语、城市和地名。智能家居里可能是“客厅空调、主卧灯带”，客服质检里可能是“续保、保单号、免赔额”，硬件售后里可能是具体型号。热词越贴近当前上下文，收益越高；热词过多或过泛，反而会干扰普通词识别。

## 标点与文本规整

ASR 原始输出可能没有标点，数字、日期、金额也可能是口语形式。后处理负责把文本变成业务可用格式。

| 口语输入 | 规整结果 |
| --- | --- |
| 二零二六年五月十七号 | 2026-05-17 |
| 一百二十三块五 | 123.5 元 |
| 幺三八零零一三八零零零 | 13800138000 |
| 打开 A B C 模式 | 打开 ABC 模式 |

文本规整要结合业务场景。客服质检需要保留口语原貌，表单录入则需要结构化格式。

## 端侧与云端

| 方案 | 优点 | 代价 |
| --- | --- | --- |
| 端侧 ASR | 低延迟、离线、隐私好 | 模型体积和算力受限 |
| 云端 ASR | 模型大、更新快、准确率高 | 依赖网络、延迟和隐私压力 |
| 混合 ASR | 端侧快速响应，云端补强 | 架构复杂，需要结果融合 |

设备控制类短命令适合端侧或混合方案；长语音转写、会议纪要和客服质检更适合云端或私有化部署。

## 指标与测试

ASR 常用 WER 或 CER 评估。可以把它们理解成“识别结果相对标准答案错了多少”：把词或字识别错了算替换，漏掉算删除，多出来算插入。中文短命令常用 CER 更直观，长文本转写更常按词或语义片段分析。业务上还要看意图准确率，因为有些识别错误不影响指令执行，有些少一个字就会改变含义。

测试集要按场景分层：

- 安静近场。
- 远场不同距离和角度。
- 噪声场景。
- 播放干扰场景。
- 方言口音。
- 业务热词。
- 长句和短命令。

## 常见问题

| 现象 | 可能原因 | 处理 |
| --- | --- | --- |
| 开头漏字 | VAD 起点晚、唤醒后缓存不足 | 增加前置缓存 |
| 结尾漏字 | VAD 结束太快 | 增加尾部静音 |
| 专有名词错 | 热词不足、领域数据少 | 加热词、补语料、做后处理 |
| 噪声下错字多 | SNR 低、NS 失真 | 调前处理、增强测试集 |
| 中间结果抖动 | 流式解码正常修正 | UI 区分 partial/final |

## 总结

ASR 不是孤立模型。输入音频质量、端点检测、热词、标点、文本规整和业务容错共同决定最终体验。工程排查时先固定录音样本，再对比前处理输出、模型输出和后处理结果。
