# 降噪与前处理

前处理位于采集链路和语音算法之间，目标是把复杂环境中的原始音频整理成更适合唤醒、识别或通话的语音。它通常包括回声消除、噪声抑制、自动增益、端点检测、波束形成和去混响。

## 前处理链路

![语音前处理原理](/acoustics/preprocessing-principles.svg)

实际顺序会因产品不同而调整。带播放的设备通常优先做 AEC，因为系统自己的声音会严重干扰后续模块。

## AEC 回声消除

AEC 用播放参考信号估计麦克风里出现的回声，再从麦克风信号中减掉。它解决的是“设备自己播放的声音被麦克风收进去”的问题。

它的原理接近“已知扬声器刚才放了什么，再估计这段声音经过外壳、桌面和房间反射后会怎样进入麦克风”。估计出来的回声越接近真实回声，抵消效果越好。播放参考缺失、延迟漂移或扬声器失真，都会让这个估计失准。

常见难点：

| 难点 | 影响 |
| --- | --- |
| 播放参考不完整 | 系统提示音、蓝牙音频或第三方播放器没有进入参考 |
| 延迟漂移 | 回声路径估计对不上，ERLE 下降 |
| 扬声器非线性 | 大音量失真无法被线性滤波器完全抵消 |
| 双讲 | 近端人声和远端播放同时存在，容易误消人声 |
| 结构漏音强 | 回声能量过大，后级算法压力增加 |

评估 AEC 时不要只听“有没有回声”，还要看近端人声是否被压制。双讲场景是关键测试项。

## NS 噪声抑制

NS 用于压制环境噪声，例如风扇、空调、车噪、人群背景声、键盘声。传统方法依赖噪声估计和频谱减法，现代系统常用深度学习模型。

可以把 NS 理解成“在每个时间片和频率片上判断更像语音还是更像噪声”。稳态噪声容易被识别，竞争人声最难，因为它和目标说话人的频谱、节奏都很像。

| 噪声类型 | 处理难度 | 说明 |
| --- | --- | --- |
| 稳态噪声 | 较低 | 空调、风扇、底噪 |
| 非稳态噪声 | 较高 | 键盘、关门、餐厅噪声 |
| 竞争人声 | 很高 | 其他人说话和目标人声频谱接近 |
| 风噪 | 很高 | 低频强、随机性高，结构防风很重要 |

过度降噪会产生“水声”“金属声”“断字”。语音识别场景中，目标不是让人听起来最干净，而是保留对识别有用的语音特征。

## AGC 自动增益

AGC 自动调节语音响度，让近距离和远距离说话都保持可用电平。

AGC 的风险：

- 把背景噪声一起放大。
- 增益变化太快，导致语音忽大忽小。
- 和 VAD 配合不好，静音段被拉高。
- 前级已经削顶时，后级 AGC 无法恢复失真。

AGC 适合补偿距离和说话音量差异，不适合修复硬件增益设计错误。

## VAD 端点检测

VAD 判断“当前是不是有人说话”，常用于唤醒、ASR 截断、录音切片和降噪控制。

关键参数：

| 参数 | 含义 |
| --- | --- |
| 起始阈值 | 判断语音开始的敏感度 |
| 结束阈值 | 判断语音结束的敏感度 |
| 前置缓存 | 语音开始前保留一点音频，避免吞字 |
| 尾部静音 | 语音结束后等待多久再截断 |

VAD 太敏感会把噪声当人声，太保守会漏掉轻声和开头。语音助手通常需要保留前置缓存，避免唤醒后第一字丢失。

## 波束形成

波束形成利用麦克风阵列增强目标方向声音、抑制其他方向声音。它需要可靠的阵列几何、通道同步和声源方向估计。

常见方式：

- 固定波束：预设几个方向，选择能量或置信度最高的方向。
- 自适应波束：根据噪声和目标位置动态调整。
- 神经网络波束形成：结合模型估计掩码或空间特征。

波束形成对混响和多个说话人比较敏感。阵列孔位或声道顺序错了，调参数通常无效。

## 去混响

混响是声音在房间多次反射后的叠加，会让语音拖尾、辅音模糊。会议室、厨房、车内和空旷大厅都会出现不同程度的混响。

处理思路：

- 结构和环境上减少强反射。
- 麦克风靠近目标声源。
- 使用阵列增强直达声。
- 使用去混响算法压制晚期反射。

去混响会影响自然度，不应盲目拉满。对 ASR 来说，轻度混响通常可接受，强混响会显著增加漏字和错字。

## 模块组合建议

| 场景 | 推荐组合 |
| --- | --- |
| 近场单麦识别 | 高通滤波 + NS + AGC + VAD |
| 远场智能音箱 | AEC + 阵列校准 + BF + NS + AGC + VAD |
| 会议通话 | AEC + 双讲保护 + NS + AGC + 去混响 |
| 车载语音 | AEC + BF + 路噪抑制 + VAD + 唤醒模型适配 |
| 耳机通话 | 多麦降噪 + 风噪检测 + AGC + 通话编码适配 |

## 排错闭环

1. 保存原始多麦音频、播放参考、前处理输出和最终送 ASR 的音频。
2. 对齐时间戳，确认是否丢帧、错帧或采样率不一致。
3. 分别关闭 AEC、NS、AGC、BF，比较指标和主观听感。
4. 用固定测试集回放，避免现场口头感受误导。
5. 对关键场景记录 SNR、ERLE、唤醒率、WER 和延迟。

## 总结

前处理不是越多越好。每个模块都可能引入失真、延迟或误判。工程上应先确定目标场景和指标，再按问题加入模块，并用原始录音和回放测试验证每次改动。
